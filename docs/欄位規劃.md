## TABLE: 規格文件

### 欄位描述
1. 規格文件編號(pk) - *主鍵 (Primary Key)*
    - 描述: 每份規格文件的唯一標識符。
    - 數據類型: 整數
    - 用途: 用於保證每份規格文件的獨特性和識別。

2. 產品型號(ak-1) - *外鍵/備用鍵 1 (Foreign Key/Alternate Key 1)*
    - 描述: 表示產品的型號號碼。
    - 數據類型: 字符串
    - 用途: 結合發行版本，用於唯一識別產品規格。

3. 發行版本(ak-1) - *備用鍵 1 (Alternate Key 1)*
    - 描述: 表示產品規格的發行版本。
    - 數據類型: 整數
    - 用途: 用於區分同一產品型號的不同規格版本。

4. 啟用時間戳
    - 描述: 記錄規格數據啟用的具體時間和日期。
    - 數據類型: DateTim
    - 用途: 用於追蹤規格文件的生效時間，幫助理解其生命週期。

5. 停用時間戳
    - 描述: 記錄規格數據停用的具體時間和日期。
    - 數據類型: DateTime
    - 用途: 用於檢視規格文件的失效時間，有助於管理產品的有效期。

6. 最後編輯時間戳
    - 描述: 記錄規格文件最後被編輯的時間。
    - 數據類型: DateTim
    - 用途: 提供對規格文件變更歷史的追蹤，便於審計和紀錄。

7. 建檔時間戳(自動)
    - 描述: 記錄規格文件在系統中創建的時間。
    - 數據類型: DateTime
    - 用途: 為規格文件的創建提供時間上的記錄，用於審計和管理。

## TABLE: 子物件

### 欄位描述
1. 子物件編號(pk) - *主鍵 (Primary Key)*
    - 描述: 每個子物件在表中的唯一識別碼。
    - 數據類型: 整數
    - 用途: 用於唯一標識每個子物件，並作為資料庫索引。

2. 規格文件編號(fk) - *外鍵 (Foreign Key)*
    - 描述: 關聯到特定產品規格文件的編號。
    - 數據類型: 整數
    - 用途: 用於連接到相關的產品規格文件，建立數據間的關聯。

3. 物件種類
    - 描述: 定義子物件的具體類型或分類。
    - 數據類型: 字符串
    - 用途: 提供子物件的分類信息，有助於理解和管理各類子物件。

4. 同種類物件識別號
    - 描述: 同一類型子物件中的序號，用於區分相同種類的多個子物件。
    - 數據類型: 整數
    - 用途: 在相同子物件種類下進行更細致的分類和識別。

5. 功能說明
    - 描述: 子物件功能。
    - 數據類型: 字符串
    - 用途: 識別子物件功能/作用。

6. 備註
    - 描述: 對子物件進行補充說明。
    - 數據類型: 字符串
    - 用途: 提供對子物件的具體描述，幫助用戶理解其用途和功能。

## TABLE: 物件群組功能映射

### 欄位描述
1. 功能組編號(pk) - *主鍵 (Primary Key)*
    - 描述: 每個分組識別資訊的唯一標識符。

## TABLE: 常數定義/靜態描述(層級: 規格文件)

### 欄位描述
1. 常數編號(pk) - *主鍵 (Primary Key)*
    - 描述: 每個規格常數項目的唯一標識符。
    - 數據類型: 整數
    - 用途: 用於唯一識別每個規格常數項目。

2. 規格文件編號(fk-1) - *外鍵 (Foreign Key)*
    - 描述: 關聯到特定產品規格文件的編號。
    - 數據類型: 整數
    - 用途: 建立常數項目與其對應的產品規格文件之間的聯繫。

3. 欄位名稱
    - 描述: 欄位的名稱或標籤。
    - 數據類型: 字符串
    - 用途: 用於展示和識別欄位的易讀名稱。

4. 欄位分類
    - 描述: 常數項目的分類，例如輸入電壓、輸出電壓等。
    - 數據類型: 字符串
    - 用途: 幫助對常數進行分類和組織。

5. 數值分類
    - 描述: 用於進一步識別常數值的特性或屬性，如最低值、中心值等。
    - 數據類型: 字符串
    - 用途: 提供更多關於常數值的描述信息。

6. 功能組編號(fk-2) - *外鍵 (Foreign Key)*
    - 描述: 指向特定分組識別資訊彙整的編號。

9.  單位

    - 數據類型: 字符串
    - 用途: 說明常數值的計量單位。

10. 數值特性
    - 描述: 關於常數值的附加特性描述，如AC、DC等。
    - 數據類型: 字符串
    - 用途: 提供常數值的額外信息，幫助理解其應用環境或條件。

11. 備註

    - 數據類型: 字符串

## TABLE: 產品測試通過條件

### 欄位描述
1. 通過條件編號(pk) - *主鍵 (Primary Key)*
    - 描述: 每個通過條件的唯一標識符。
    - 數據類型: 整數
    - 用途: 用於唯一識別每個通過條件項目。

2. 規格文件編號(fk-1) - *外鍵 (Foreign Key 1)*
    - 描述: 關聯到特定產品規格文件的編號。
    - 數據類型: 整數
    - 用途: 建立通過條件與其對應的產品規格文件之間的聯繫。

3. 欄位名稱
    - 描述: 通過條件項目的名稱或標籤。
    - 數據類型: 字符串
    - 用途: 提供通過條件項目的易讀名稱，方便識別和參考。

4. 欄位分類
    - 描述: 通過條件項目的分類，例如輸入電流、輸出電壓等。
    - 數據類型: 字符串
    - 用途: 幫助對通過條件進行分類和組織。

5. 功能組編號(fk-2) - *外鍵 (Foreign Key 2)*
    - 描述: 指向特定分組識別資訊彙整的編號。
    - 數據類型: 整數
    - 用途: 建立通過條件與相關分組的聯繫。

6. 設定輸入電壓狀態
    - 描述: 描述測試中使用的輸入電壓狀態。
    - 數據類型: 字符串
    - 用途: 指定測試條件下的輸入電壓狀態。

7. 特殊輸入電壓值
    - 描述: 當設定特殊輸入電壓時所使用的具體電壓值。
    - 數據類型: 數值
    - 用途: 提供特殊測試條件下的輸入電壓數值。

8. 設定負載電流狀態
    - 描述: 描述測試中使用的負載電流狀態。
    - 數據類型: 字符串
    - 用途: 指定測試條件下的負載電流狀態。

9. 特殊負載電流值
    - 描述: 當設定特殊負載電流時所使用的具體電流值。
    - 數據類型: 數值
    - 用途: 提供特殊測試條件下的負載電流數值。

10. 設定環境溫度狀態
    - 描述: 描述測試中使用的環境溫度狀態。
    - 數據類型: 字符串
    - 用途: 指定測試條件下的環境溫度狀態。

11. 特殊環境溫度值
    - 描述: 當設定特殊環境溫度時所使用的具體溫度值。
    - 數據類型: 數值
    - 用途: 提供特殊測試條件下的環境溫度數值。

12. 負載類型
    - 描述: 描述測試中使用的負載類型。
    - 數據類型: 字符串
    - 用途: 指定測試條件下的負載類型。

13. 下限
    - 描述: 測試條件的最低允許值。
    - 數據類型: 數值
    - 用途: 確定測試條件的下限值。

14. 上限
    - 描述: 測試條件的最高允許值。
    - 數據類型: 數值
    - 用途: 確定測試條件的上限值。

15. 單位
    - 描述: 測試條件值的計量單位。
    - 數據類型: 字符串
    - 用途: 提供測試條件值的度量標準。

16. 數值特性
    - 描述: 關於測試條件值的附加特性描述。
    - 數據類型: 字符串
    - 用途: 提供測試條件值的額外信息，幫助理解其應用環境或條件。

## FORMULA: 換算公式

### 輸出電壓準確度
- $\text{Output Voltage Accuracy} = \frac{V_{out}@V_{inN}@L_{full} - V_{out(th)}}{V_{out(th)}} \times 100\%$
- V<sub>out</sub>@V<sub>inN</sub>@L<sub>full</sub> : 當輸入電壓等於產品規格的額定值，且輸出負載等於產品規格的滿載值時，實際量測到的輸出電壓值。
- V<sub>out(th)</sub> : 產品規格中的輸出電壓值。

### 線調整率（有穩壓）
- $\text{Line Regulation} = \frac{V_{out}@V_{inH} - V_{out}@V_{inL}}{V_{out}@V_{inL}} \times 100\%$
- V<sub>out</sub>@V<sub>inH</sub> : 當輸入電壓等於產品規格的上限值時，實際量測到的輸出電壓值。
- V<sub>out</sub>@V<sub>inL</sub> : 當輸入電壓等於產品規格的下限值時，實際量測到的輸出電壓值。

### 線調整率（無穩壓）
- $\text{Line Regulation} = \frac{(V_{out}@V_{inH} - V_{out}@V_{inL}) / V_{out}@V_{inL}}{(V_{inH(th)} - V_{inL(th)}) / V_{inL(th)}} \times 100\%$
- V<sub>out</sub>@V<sub>inH</sub> : 當輸入電壓等於產品規格的上限值時，實際量測到的輸出電壓值。
- V<sub>out</sub>@V<sub>inL</sub> : 當輸入電壓等於產品規格的下限值時，實際量測到的輸出電壓值。
- V<sub>inH(th)</sub> : 產品規格中的輸入電壓上限值。
- V<sub>inL(th)</sub> : 產品規格中的輸入電壓下限值。

### 負載調整率
- $\text{Load Regulation} = \frac{V_{out}@L_{light} - V_{out}@L_{full}}{V_{out}@L_{full}} \times 100\%$
- V<sub>out</sub>@L<sub>light</sub> : 當輸出負載等於產品規格的輕載值時，實際量測到的輸出電壓值。（ps. 部分產品輕載值=0）
- V<sub>out</sub>@L<sub>full</sub> : 當輸出負載等於產品規格的滿載值時，實際量測到的輸出電壓值。

### 輸出電壓平衡率
- $\text{Output Voltage Balance} = 2 \times \frac{\vert V_{out+}@V_{inN} \vert - \vert V_{out-}@V_{inN} \vert}{\vert V_{out+}@V_{inN} \vert + \vert V_{out-}@V_{inN} \vert} \times 100\%$
- V<sub>out+</sub>@V<sub>inN</sub> : 當輸入電壓等於產品規格的額定值時，實際量測到的正向輸出電壓值。
- V<sub>out-</sub>@V<sub>inN</sub> : 當輸入電壓等於產品規格的額定值時，實際量測到的負向輸出電壓值。

### 效率
- $\text{Efficiency} = \frac{P_{out}@L_{full}}{P_{in}@L_{full}} = \frac{\sum (V_{out}@L_{full} \times I_{out(th)}@L_{full})}{V_{in(th)}@L_{full} \times I_{in}@L_{full}} \times 100\%$
- Vin Options : @ { Low / Nominal / High } Input Voltage
- P<sub>out</sub>@L<sub>full</sub> : 當輸出負載等於產品規格的滿載值時，實際量測到的輸出功率值。
- P<sub>in</sub>@L<sub>full</sub> : 當輸出負載等於產品規格的滿載值時，實際量測到的輸入功率值。
- V<sub>out</sub>@L<sub>full</sub> : 當輸出負載等於產品規格的滿載值時，實際量測到的輸出電壓值。
- V<sub>in(th)</sub>@L<sub>full</sub> : 當輸出負載等於產品規格的滿載值時，產品規格中的輸入電壓值。
- I<sub>out(th)</sub>@L<sub>full</sub> : 當輸出負載等於產品規格的滿載值時，產品規格中的輸出電流值。
- I<sub>in</sub>@L<sub>full</sub> : 當輸出負載等於產品規格的滿載值時，實際量測到的輸入電流值。

### 啟動過衝率
- $\text{Percentage Overshoot} = \frac{V_{out-Peak}@Startup - V_{out(th)}}{V_{out(th)}} \times 100\%$
- V<sub>out-Peak</sub>@Startup : 在產品啟動階段，實際量測到的輸出電壓峰值。
- V<sub>out(th)</sub> : 產品規格中的輸出電壓值。

### 暫態變動率
- $\text{Transient Variation Rate} = \frac{V_{out-Peak / Valley}-V_{out-Balance}}{V_{out(th)}} \times 100\%$
- 公司自訂義的公式，學術上似乎沒有使用。chatGPT提到的暫態變動率=(峰值or谷值-平衡電壓)/暫態回復時間。
- V<sub>out-Peak or Valley</sub> : 測試暫態響應的過程中，當負載電流突然變化(209表目前有兩種 : 75~100% or 50~100%)後，實際量測到的輸出電壓峰值or谷值。(具同仁說會用測試暫態響應的視波器的圖，點圖取3次波峰。猜測是取平均的波峰值。)
- V<sub>out-Balance</sub> : 測試暫態響應的過程中，於視波器設定的穩定輸出電壓值。（未精確定義如何取這個穩定輸出電壓值，大多選視波器內第一個負載從75%變化到100%後穩定的電壓值做基準線。）
- V<sub>out(th)</sub> : 產品規格中的輸出電壓值。(工程部替代料Excel公式使用輸出電壓理論值，其實上方的V<sub>out-Balance</sub>相對更適合，然而這個參數僅於ATS內用於計算出暫態變動量，並未被保存到原始資料參數內。)
- V<sub>out-Peak or Valley</sub> - V<sub>out-Balance</sub> : 工程部替代料ATS匯出結果的原始數據: "暫態變動量(mV)"。(工程部替代料ATS匯出結果欄位名被錯誤標記為以下格式: "FL暫態(%)(+)")

#### 解釋
S-I/P V-MIN, S-I/P V-NON, S-I/P V-HIGH：這些可能代表輸入電壓(Input Voltage)的最小值、正常值和最高值的測試。這類測試確保設備在不同輸入電壓條件下能正常工作。

S-O/P1 V, S-O/P2 V：代表第一輸出(Output 1)和第二輸出(Output 2)的電壓值。

S-O/P1 I, S-O/P2 I：代表第一輸出和第二輸出的電流值。

S-EFF_MIN：最小效率測試，測量設備運行時的最低效率。

S-I/P I@NOLOAD, S-O/P1 V@NOLOAD, S-O/P2 V@NOLOAD：空載時的輸入電流、第一輸出和第二輸出的電壓。

S-REFL.CURRENT, S-I/P I@MAX.：反射電流和最大輸入電流的測量。

S-O/P V-BLANCE, S-O/P1 V-LOAD REG., S-O/P2 V-LOAD REG.：輸出電壓平衡、輸出1和輸出2的負載調節率。

S-O/P1 V-LINE REG., S-O/P2 V-LINE REG.：輸出1和輸出2的線路調節率。

S-O/P1 R&N, S-O/P2 R&N：輸出1和輸出2的紋波和噪聲。

S-TRANSIENT RT, S-TRANSIENT DV：瞬態響應時間和瞬態偏差。

S-SHORT CURRENT@VIN, S-OVERLOAD@MAX, etc.：輸入短路電流、最大負載下的過載保護。

S-REMOTE-ON@MAX, S-REMOTE-OFF@MAX, etc.：遠程開關機測試在最大條件下的行為。

S-HIPOT, S-IR, S-ISOCAP：高壓絕緣測試(High Potential Test)、絕緣電阻(Insulation Resistance)和絕緣容量測試。

S-FREQ.@MAX, S-FREQ.@MIN：在最大和最小頻率下的性能測試。
